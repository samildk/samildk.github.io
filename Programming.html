Human-in-the-Loop Pair Verification System

Developed a human-in-the-loop system to verify the similarity of image pairs where the AI gave uncertain or false results after Milvus clustering. This ensures 100% accuracy in cluster assignments for visually similar or dissimilar images.

Workflow Overview

  • Flagging AI-Uncertain Pairs: Identify pairs of images where AI predictions are unreliable.
  • Human Verification Interface: Present side-by-side comparison for human operators to mark same or different.
  • Final Correction: Human decision overrides AI result and ensures cluster consistency.
  • Database Update: Update database with verified results and adjust cluster assignments accordingly.

Technical Highlights

  • Human-in-the-loop verification for AI-ambiguous image pairs.
  • Post-clustering review to fix false positives or uncertain similarity results.
  • Flask-based web interface for fast, intuitive human review.
  • Database integration ensures consistency and reliability of clusters.
  • Scalable pipeline supporting large-scale flagged pairs efficiently.

End-to-End Automated Multimedia Ad Clustering System

I developed a full end-to-end system for processing single-image ads and short videos, combining keyframe extraction, histogram-based similarity, Milvus vector database clustering, and AI-driven visual verification. The system uses a modular Python code design and a Flask API to query a fine-tuned Qwen model for high-precision verification.

Pipeline Overview

  • Video & Image Ingestion: Efficiently reads new multimedia content and prepares it for analysis.
  • Keyframe Extraction & Noise Handling: Generates soft histograms, detects duplicates, merges noisy segments, and extracts representative frames.
  • Database Storage & Foldering: Stores keyframes, histograms, and metadata in PostgreSQL and organizes files in structured folders.
  • Feature Vector Generation: Converts keyframes into dhash vectors for fast similarity search in Milvus.
  • Milvus-Based Clustering: Assigns cluster IDs using nearest-neighbor search; dynamically reuses or creates new clusters.
  • Qwen Verification: Uses Flask API serving a fine-tuned Qwen model to confirm visual similarity before final assignment.
  • Final Cluster Assignment & Update: Updates database cluster IDs and moves content to organized folders.

Technical Highlights

  • Modular Python code structure for ingestion, preprocessing, clustering, and verification.
  • AI verification via fine-tuned Qwen model served through Flask API.
  • Milvus vector search with L2 distance for scalable clustering.
  • Noise handling and edge case management for overlapping or partial videos.
  • Transactional PostgreSQL database integration for reliability.
  • Parallel and scalable pipeline supporting real-time multimedia ingestion.

Impact & Use Case

  • Automates clustering and organization of single-image ads and short videos.
  • Detects near-duplicate content with high precision using combined vector and AI verification.

Automated Video Keyframe Extraction and Multimodal Tizer Categorization

I developed a high-performance system for automated video analysis and categorization, designed to extract, process, and match video keyframes for large-scale advertising assets, known internally as tizers. The project transforms raw video streams into structured datasets of keyframes, histograms, and semantic categories for analytics, duplicate detection, and recommendation systems.

Pipeline Overview

  • Keyframe Extraction: Reads video frames using OpenCV or FFmpeg, generates soft-normalized grayscale histograms, compares frames with Bhattacharyya distance, and selects keyframes.
  • Representative Frame Identification: Finds frames with maximum representative length and highest entropy; supports dual max-frame detection for multiple key segments.
  • Histogram-Based Matching: Compares keyframes against previously processed tizers to detect duplicates or near-duplicates using histogram similarity metrics.
  • Category Assignment: Assigns videos to predefined or generated categories using entropy and histogram heuristics; groups related tizers under a “mother tizer.”
  • Database Integration: Stores keyframes and metadata in PostgreSQL tables (Tbl_TizerHistogram3, Tbl_MatchHist, Tbl_Unique) using SQLAlchemy with batch and transactional operations; supports multiprocessing for parallel processing.

Technical Highlights

  • Multimodal Video Processing with representative frames and soft histograms.
  • Duplicate Detection & Matching using histogram comparison and Bhattacharyya distance.
  • Category Mapping & Mother Tizer Assignment for hierarchical organization.
  • Parallel & Scalable Processing via Python multiprocessing.
  • Database-Centric Design with PostgreSQL transactional safety and batch updates.
  • Robust Frame Representation including frame number, start time, length, normalized histogram, and entropy.
  • FFmpeg & OpenCV integration for multi-format and high-resolution video processing.

Impact & Use Case

  • Automates creation of structured keyframe datasets from hundreds of hours of video content.
  • Enables robust duplicate detection, similarity ranking, and content categorization.
  • Supports hierarchical content mapping for analytics, reporting, and recommendations.
  • Handles large-scale operations with fault-tolerant parallel processing and transactional database safety.

Technologies Used

  • Programming & Libraries: Python, OpenCV, NumPy, PIL, SciPy, subprocess
  • Database & ORM: PostgreSQL, SQLAlchemy
  • Video Tools: FFmpeg, ffprobe
  • Parallelism & Performance: Python multiprocessing, concurrent frame processing
  • Algorithms & Metrics: Soft histogram smoothing, Bhattacharyya distance, entropy calculation, frame-to-frame similarity

Fine-Tuning Qwen2.5-VL for Multimodal Image Similarity Reasoning

I built a full supervised fine-tuning pipeline for Qwen2.5-VL, an open-source multimodal vision-language model, to improve its performance on image-pair reasoning tasks. The model is trained to analyze two images simultaneously and answer whether they are visually or semantically the same using a controlled Yes/No response.

This project focuses on adapting a large vision-language foundation model for a specialized real-world task: binary visual similarity judgment. I designed the system to process paired image inputs, apply Qwen’s multimodal chat formatting, correct noisy labels, and train the model using efficient fine-tuning techniques.

To make fine-tuning practical and scalable, I used LoRA adapters with 4-bit quantization, allowing the model to be adapted efficiently without updating all base model parameters. I also implemented a custom weighted loss function to emphasize difficult or incorrectly answered examples, helping the model learn from its previous mistakes while preserving correct behavior on reliable samples.

Key Features

  • Two-Image Multimodal Training: Model receives two images in the same prompt and learns to compare them directly.
  • Custom Dataset Pipeline: Reads JSONL examples, validates image-pair structure, loads images, cleans prompts, prepares multimodal tensors, and masks labels.
  • Label Correction Logic: Automatically flips Yes/No answers for incorrect examples to provide corrected supervision.
  • Weighted Loss for Hard Examples: Emphasizes incorrect examples to help the model learn from mistakes.
  • Efficient Fine-Tuning: Uses LoRA and 4-bit quantization to reduce GPU memory usage and adapt efficiently.
  • Baseline and Post-Training Evaluation: Measures generation accuracy before and after fine-tuning for measurable improvements.
  • Diagnostic Evaluation Modes: Separately measures correction of wrong examples and preservation of correct answers.
  • Inference and Batch Prediction: Supports single image-pair prediction and batch prediction from CSV/JSONL files.

Technologies Used

  • Models and AI Frameworks: Qwen2.5-VL, Hugging Face Transformers, PEFT, LoRA, BitsAndBytes
  • Deep Learning: PyTorch, causal language modeling, custom loss functions, gradient accumulation
  • Multimodal Processing: PIL, Qwen VL utilities, two-image prompt construction
  • Training Optimization: 4-bit quantization, NF4, paged AdamW 8-bit, cosine scheduler, early stopping
  • Evaluation: validation accuracy, label-specific diagnostics, deterministic generation, Yes/No constrained decoding
  • Data Formats: JSONL, CSV, image-pair datasets

Data Analysis and Customer Segmentation for E-commerce

Project Overview

This project is a comprehensive analysis of e-commerce transaction data, focusing on user behavior, product interactions, and customer segmentation. It was part of my collaboration with DigiNext to analyze large datasets and derive actionable insights for improving customer engagement, sales strategies, and product recommendations.

Data Cleaning and Preparation

Objective: Clean and preprocess raw transaction data.

Steps:

  • Handled missing and duplicate data.
  • Removed negative values in transaction quantity and ensured consistency.
  • Converted timestamps to appropriate formats for easier analysis.

Feature Engineering

Objective: Extract meaningful features from transaction data.

Key Features:

  • Extracted time features from the InvoiceDate column, such as hour, day of week, day of month, and week.
  • Calculated the event score based on event types like purchase, view, and cart.

Exploratory Data Analysis (EDA)

Objective: Gain a deeper understanding of the data and identify key trends.

Findings:

  • Customer Purchase Trends: Analyzed the number of transactions and revenue trends by day, week, and month using bar plots and time-series analysis.
  • Product Insights: Visualized product popularity by category, price distribution, and customer interest.
  • Customer Segmentation: Grouped customers based on their total purchase amount and quantity using K-Means clustering, identifying four distinct clusters.

Statistical Analysis & A/B Testing

Objective: Test hypotheses related to group performance (Group A vs. Group B).

Hypothesis: Investigated whether there is a significant difference in the average purchase amounts between two groups of users.

Method: Conducted a t-test to compare the means of the two groups, with results showing no significant difference in the average purchase amount.

Customer Segmentation

Objective: Cluster customers based on their purchasing behavior to identify high-value customers.

Method: Used K-Means clustering to segment customers into four clusters based on total purchase amounts and quantities purchased.

Outcome: Identified clusters with very high purchase amounts (Cluster 2) and clusters with smaller, less frequent purchases (Cluster 0).

Key Insights & Impact

  • Customer Behavior: The analysis revealed clear differences in purchase behavior across users. High-value customers were identified, with their purchasing patterns being leveraged for targeted marketing.
  • Revenue Growth Opportunities: By segmenting customers, I helped identify strategies to boost conversions and target high-value customers more effectively.
  • Improved Product Recommendations: Insights from customer interactions with products enabled more targeted product recommendations, increasing user engagement.
  • Efficient Resource Allocation: Understanding time-of-day trends allowed the business to optimize staffing, promotions, and inventory management.

Technologies Used

Python (Pandas, NumPy, Matplotlib, Seaborn)

Data Cleaning & Transformation: Pandas, NumPy

Statistical Analysis: T-test (SciPy)

Clustering: K-Means (Scikit-learn)

Visualization: Matplotlib, Seaborn
